Valkyrien Portals — Changelog
=============================

1.0.0 (Beta) — 2026-06-22
-------------------------
First public Beta. Compatibility patch for Valkyrien Skies 2 and Immersive Portals on
Forge 1.20.1 (Minecraft 1.20.1).

Fixes
  - Boot crash: cancels Valkyrien Skies' redundant Frustum dead-loop mixin (via a MixinSquared
    MixinCanceller) so it no longer collides with Immersive Portals' Frustum overwrite. This is
    the core fix that lets the two mods load together at all.
  - Portal camera: bridges Immersive Portals' nested-pass camera matrix through the cull-frustum
    path so terrain culls correctly when the scene is viewed through a portal.
  - Portal fog: during a nested portal pass, Embeddium's fog-occlusion search distance ignores the
    host pass's collapsed shader fog, restoring the full render distance through portals.
  - Dimension transit: tolerates the ClassCastException Valkyrien Skies otherwise throws on every
    portal transit when Immersive Portals owns the client chunk source (ImmPtlClientChunkMap),
    while preserving VS's player-side known-ship cleanup. Best-effort / non-required: on a Valkyrien
    Skies build whose internal lambda index differs from 2.4.11 the guard quietly no-ops rather than
    failing to load.

Known issues
  - Valkyrien Skies ships sitting in a remote dimension are not yet rendered when viewed through a
    portal — this needs server-side ship tracking for portal-viewed dimensions. See KNOWN_ISSUES.md.

Requires
  - Valkyrien Skies 2 (>= 2.0)
  - Immersive Portals (>= 3.0)
  - MixinSquared (bundled inside Supplementaries, or installed standalone)
  - An Embeddium / Sodium-based renderer (the ship render path this mod bridges)
